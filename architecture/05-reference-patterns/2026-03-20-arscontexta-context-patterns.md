## Patterns Worth Keeping

Useful invariant:
- persistent self/methodology space
- knowledge graph space
- operational queue/session space

Directive implication:
- keep clear separation between doctrine, artifact knowledge, and operational state

### Phase pipeline

Useful invariant:
- work moves through named phases with explicit transformation intent

Directive implication:
- strengthens how Discovery, Architecture, and Forge define handoffs and artifact mutation

### Schema-backed quality

Useful invariant:
- generated or written notes are checked against structural expectations

Directive implication:
- aligns with Directive contract-first artifact handling

### Fresh context per phase

Useful invariant:
- each phase should not inherit all previous context blindly

Directive implication:
- supports bounded slices, explicit handoffs, and anti-context-bloat workflow design
