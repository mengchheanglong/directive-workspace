# Runtime Follow-up

This folder stores Runtime-facing follow-up artifacts that originate from:
- Discovery routing
- Architecture handoff
- Runtime planning and runbooks

These files are Runtime lane-owned workflow artifacts.

Mission Control may execute the runtime path, but the follow-up record belongs here.

Deferred follow-up rule:
- every deferred Runtime follow-up must include a formal re-entry contract and clear preconditions before experiment.

Template:
- `shared/templates/runtime-follow-up-record.md`
